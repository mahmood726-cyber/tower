# REPRODUCE.md

## Reproduction Guide for TruthCert/Burhan Evidence System

This document explains how to reproduce the evidence capsules generated by this system.

---

## Reproduction Modes (per Spec v0.8 Section 21)

### Public Mode (Recommended for Verification)

Public mode uses only:
- Open registry data (CT.gov/AACT, ICTRP)
- Open access publications (PubMed, PMC)
- Public context data (WHO, World Bank)
- Locators + hashes for any restricted sources

**Steps:**

```bash
# 1. Clone the repository
git clone https://github.com/example/tower.git
cd tower

# 2. Install dependencies
pip install -r requirements.txt

# 3. Verify a capsule
./scripts/capsule_verify.sh --capsule capsules/2026-02-07/cardiology-sglt2i/

# 4. Reproduce the computation
./scripts/capsule_reproduce.sh --capsule capsules/2026-02-07/cardiology-sglt2i/ --mode public
```

**What gets verified:**
- All witness hashes match stored content
- All validators produce same results
- Badge computation is deterministic
- Drift events are reproducible

### Full Mode (Local Validation)

Full mode requires local copies of restricted sources to validate locators.

**Prerequisites:**
- Access to full-text publications (via institutional access)
- Local storage for retrieved content

**Steps:**

```bash
# 1. Set up full-text retrieval credentials
export PUBMED_API_KEY="your-key"

# 2. Retrieve restricted sources locally
./scripts/source_retrieve.sh --capsule capsules/2026-02-07/cardiology-sglt2i/

# 3. Verify locators match hashes
./scripts/capsule_verify.sh --capsule capsules/2026-02-07/cardiology-sglt2i/ --mode full

# 4. Full reproduction
./scripts/capsule_reproduce.sh --capsule capsules/2026-02-07/cardiology-sglt2i/ --mode full
```

**Note:** Restricted content is never redistributed. We store only locators + hashes.

## Verification Checklist

For any capsule, verify:

1. **MANIFEST.json integrity**
   ```bash
   # Capsule ID should match sha256 of manifest content
   ./scripts/verify_capsule_id.sh --manifest MANIFEST.json
   ```

2. **Witness chain**
   ```bash
   # All witnesses should have valid locators and hashes
   ./scripts/verify_witnesses.sh --capsule ./
   ```

3. **Validator results**
   ```bash
   # Re-run validators and compare
   ./scripts/rerun_validators.sh --capsule ./
   ```

4. **Badge computation**
   ```bash
   # Badge should be deterministically reproducible
   ./scripts/compute_badge.sh --capsule ./
   ```

## Troubleshooting

### Hash Mismatch
If a witness hash doesn't match:
1. Check if the source has been updated (registry edit)
2. Compare timestamps
3. This should be recorded as a DriftEvent

### Validator Disagreement
If validators produce different results:
1. Check spec version compatibility
2. Verify input data is identical
3. Report as a potential bug

### Missing Witnesses
If witnesses are missing:
1. Check if source is restricted
2. Use locator to retrieve
3. Verify hash matches
